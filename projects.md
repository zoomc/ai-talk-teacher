# SpeakFlow — AI 口语练习应用

> 跨平台 AI 英语口语练习应用
> 更新日期：2026-06-28
> 技术栈：Flutter（Dart）
> 目标平台：macOS · Web · iOS · Android

---

## 一、产品定位

面向英语学习者的 AI 口语练习应用。用户与 AI 虚拟外教进行自然英语对话，AI 在对话中实时纠正语法和用词错误，自动记录错误，通过复习模式巩固提升，形成**完整学习闭环**。

**核心差异点（对比市面产品）：**
- **学习闭环**：练习→自动记录错误→间隔复习→巩固掌握（国内竞品普遍缺失）
- **3-Profile 系统**：LLM / STT / TTS 三套 API 独立配置，用户自带 Key，零运营成本
- **虚拟外教**：Live2D 角色 + 唇形同步，沉浸感（国内竞品均无）
- **自然纠正**：Praktika 式不打断对话流，自然重述正确表达
- **话题连续性**：随时接续上次话题
- **AI 自由度**：不绑定单一服务，兼容所有 OpenAI 协议

### 2.4 SpeakFlow 差异化定位

基于竞品分析，SpeakFlow 的核心差异点：

| 差异点 | 说明 | 对标竞品劣势 |
|--------|------|-------------|
| **完整学习闭环** | 练习→自动标记语法错误→记录→间隔复习→巩固，SM-2 算法 + AI 聊天式复习 | 流利说有复习但固定模式；可栗/咕噜无系统复习 |
| **3-Profile 系统** | LLM / STT / TTS 三套 API 独立配置，用户自带 Key，零运营成本 | 三家均绑定自家系统，用户无选择权 |
| **虚拟外教角色** | Live2D 角色 + 唇形同步，沉浸感强 | 国内三家均无虚拟人物 |
| **自然纠正策略** | Praktika 式：不打断对话流，自然重述 | 流利说靠跟读评分，可栗/咕噜实时打断式纠正 |
| **话题连续性** | 随时接续上次话题 | 三家均无此功能 |
| **云端 STT/TTS** | Deepgram/Fish Audio 等专业级识别和合成，支持非母语口音、语法错误、多语言混说 | 系统内置 STT 无法应对初学者的真实场景 |

### 3.2 对话练习模式

| 功能 | 说明 |
|------|------|
| **水平定级** | 新用户首次使用时完成定级测试（AI 引导简短对话），评估英语水平，自动设置对话难度起点 |
| **自由对话** | AI 引导自然英语聊天，自动把控话题覆盖和难度 |
| **场景练习** | 预设场景：餐厅、机场、商务会议、面试、约会等。游戏化场景卡片选择（分类展示 + 进度标记 + 难度标签） |
| **智能纠正** | 参考 Praktika 策略——不打断对话流，在回复中自然重述正确表达 |
| **纠正强度** | 三档可调：温和（偶尔纠正）/ 适中 / 严格（每次必纠） |
| **错误自动记录** | 每次纠正自动提取并存储：原文→纠正→类型（语法/用词/发音），存入错误库供复习使用 |
| **话题连续性** | 每次打开应用可选择：继续上次话题 / 开始新话题 |
| **AI 主导节奏** | LLM 通过 System Prompt 控制对话走向，确保贴近实用场景 |

### 3.3 复习模式

| 功能 | 说明 |
|------|------|
| **AI 引导复习** | 虚拟外教以"聊天式课堂"风格带你复习，语气轻松不严肃 |
| **错误回顾** | 从错误库中提取薄弱点，融入对话重新练习 |
| **间隔重复** | SM-2 算法安排复习间隔，越熟的错误出现频率越低 |
| **进度统计** | 已掌握 / 待复习 / 新增错误的数量和趋势 |

### 3.4 服务配置（3-Profile 系统）

应用依赖三类云端服务，全部由用户自行注册获取 API Key。三套 Profile 独立管理、独立切换。

```

│  │  当前：DeepSeek 主力                          │   │
│  │  模型：deepseek-v4-flash                      │   │
│  │  [ 切换 ]  [ 管理 Profiles ]                  │   │
│  └──────────────────────────────────────────────┘   │
│                                                      │
│  🎤 语音识别（STT）                                   │
│  ┌──────────────────────────────────────────────┐   │
│  │  当前：Deepgram                               │   │
│  │  状态：✅ 已连接                                │   │
│  │  [ 切换 ]  [ 管理 Profiles ]                  │   │
│  └──────────────────────────────────────────────┘   │
│                                                      │
│  🔊 语音合成（TTS）                                   │
│  ┌──────────────────────────────────────────────┐   │
│  │  当前：Fish Audio                             │   │
│  │  音色：English Female - Natural               │   │

│  └──────────────────────────────────────────────┘   │
│                                                      │
│            [ 📥 导入全部 ]  [ 📤 导出全部 ]            │
└──────────────────────────────────────────────────────┘
```

**为什么全部用云端 API？**

| 场景 | 系统内置 STT 的问题 | 云端 STT 的优势 |
|------|-------------------|----------------|
| 初学者发音不准确 | 识别率低，大量误识别 | Deepgram 等针对多口音优化 |
| 语法错误多 | 无法区分"说得对"和"说错了" | 准确转录原文，让 LLM 分析错误 |
| 中英混说 | 混合语言识别能力差 | 支持 code-switching 识别 |
| 语速不均匀 | 断句错误，丢失内容 | 实时流式识别，动态调整 |
| TTS 趣味性 | 系统语音机械、无情感 | Fish Audio/ElevenLabs 音色丰富自然 |

| 供应商 | 特点 | 计费 | 推荐度 |

| **OpenAI Whisper API** | 识别准确、兼容性好 | $0.006/分钟 | ⭐⭐ |
| **Google Cloud Speech** | 多语言强、实时流式 | $0.006-0.009/分钟 | ⭐⭐ |

**TTS Profile 支持的供应商：**

| 供应商 | 特点 | 计费 | 推荐度 |

**Profile 管理操作（LLM / STT / TTS 通用）：**

| 操作 | 说明 |
|------|------|
| 新建 | 选择供应商 → 填写 API Key → 测试连接 → 保存 |
| 测试连接 | 发送简单请求验证 Key 有效性，显示延迟和状态 |
| 保存 | 仅保存，不切换 |
| 保存并应用 | 保存 + 立即切换为当前活跃 Profile |
| 应用 | 列表中点击，即时生效 |
| 编辑 | 修改任意字段，保存后如果当前激活则自动更新 |
| 复制 | 基于现有 Profile 快速创建副本 |
| 删除 | 需确认；**不能删除当前激活的 Profile** |
| 导入/导出 | JSON 格式，三套 Profile 一起导出，Key 默认脱敏 |

**数据安全：**
- API Key 存入系统级安全存储（Keychain / EncryptedSharedPreferences）
- 不以明文写入数据库
- 导出时 Key 字段默认掩码处理
- 本地缓存 TTS 音频，避免重复调用

**核心视觉元素：**

|------|------|
| 毛玻璃背景 | 半透明磨砂材质，营造层次感和深度 |
| 微渐变 | 紫→蓝→青冷色调渐变，科技感与学习感兼具 |
| 柔光效果 | 关键元素（按钮、卡片）带柔和外发光，暗示 AI 智能感 |
| 大圆角 | 16-24px 圆角卡片，友好不生硬 |
| 深色优先 | 默认深色主题，护眼 + 科技氛围；支持浅色模式 |
| 流体动画 | 状态切换、页面转场使用弹性动画，不突兀 |

**色彩方案：**

```
主色调（深色模式）：
  背景：    #0A0E1A → #111827（深空蓝渐变）
  玻璃层：   rgba(255,255,255, 0.06) ~ 0.12
  主强调色：  #6C5CE7（紫色，AI/科技感）
  次强调色：  #00D2FF（青色，活力感）
  成功/纠正： #00E676（绿色）
  错误标注：  #FF5252（红色，用于错误词高亮）
  文字主色：  #F0F0F0
  文字次色：  #8892A4

浅色模式：
  背景：    #F5F7FB → #FFFFFF
  玻璃层：   rgba(255,255,255, 0.7) ~ 0.9
  强调色同深色模式，饱和度微调
```

### 4.2 平台适配策略

| 平台 | 风格实现 | 说明 |
|------|---------|------|
| **iOS / macOS** | Liquid Glass 风格 | 追随 iOS 26 设计语言：系统级毛玻璃材质、动态透明度、原生导航手势。使用 `BackdropFilter` + 系统材质 API |
| **Android** | Material You + 玻璃质感 | Material 3 组件为基础，叠加毛玻璃效果。支持 Dynamic Color 动态取色（壁纸提取主题色） |
| **Web** | 与所在平台风格统一 | macOS Safari 用 Liquid Glass 风格，Windows Chrome 用 Material 风格 |

**适配原则：**
- 核心组件（聊天气泡、AI 角色区域、录音按钮）视觉风格四端统一
- 导航栏、弹窗、系统交互遵循各平台原生规范
- 玻璃效果通过 `BackdropFilter`（Flutter 原生支持）实现，Web 端用 `backdrop-filter` CSS

## 五、数据架构

### 5.1 本地存储方案

| 数据类型 | 存储方式 | 说明 |
|---------|---------|------|
| **Profile 配置** | SQLite | 名称、URL、加密后的 Key、模型名 |
| **API Key** | 系统 Keychain | iOS/macOS: Keychain; Android: EncryptedSharedPreferences; Web: 加密 localStorage |
| **对话历史** | SQLite | 会话 ID、消息列表、时间戳、话题标签 |
| **错误记录** | SQLite | 原文、纠正、类型、时间、复习次数、下次复习时间 |
| **TTS 音频缓存** | 本地文件系统 | 哈希索引，避免重复合成 |
| **Viseme 时间线** | JSON 文件 | 与 TTS 音频配对，唇形同步用 |
| **用户设置** | SQLite / SharedPreferences | 非敏感配置 |

### 5.2 Profile 数据安全

参考 cc-switch 的设计：
- **原子写入**：写入临时文件 → 重命名替换，防止写入中断导致损坏
- **自动备份**：保留最近 5 份 Profile 数据备份
- **Key 加密**：API Key 存入系统级安全存储，不以明文写入数据库
- **导出脱敏**：导出 Profile 时 Key 字段默认掩码处理

## 六、技术方案概要

### 6.1 框架与关键依赖

| 层面 | 方案 |
|------|------|
| **跨平台框架** | Flutter 3.x + Dart |
| **AI 对话** | OpenAI 兼容协议，用户自选提供商 |
| **STT** | Deepgram / OpenAI Whisper / Google Cloud Speech / Azure Speech（用户自带 Key） |
| **TTS** | Fish Audio / ElevenLabs / OpenAI TTS / Azure TTS（用户自带 Key） |
| **UI 风格** | Glassmorphism + 平台自适应（iOS: Liquid Glass; Android: Material You + 玻璃质感） |
| **唇形同步** | MVP：振幅驱动嘴型；生产：Live2D + Rhubarb viseme |
| **动画引擎** | Live2D Cubism SDK / Rive / Flutter 动画 |
| **音频** | `record`（录音）+ `just_audio`（播放） |
| **数据库** | `sqflite`（SQLite） |
| **状态管理** | Riverpod 或 Bloc |
| **路由** | GoRouter |
| **加密** | `flutter_secure_storage` |
| **玻璃效果** | `liquid_glass_widgets`（iOS 26 Liquid Glass）+ `BackdropFilter` 降级 |

### 6.2 唇形同步方案

```
阶段 1 (MVP):     音频振幅 → 3 状态嘴型（闭/半开/全开）→ 2D 插画
阶段 2 (生产):     TTS 音频 → Rhubarb 离线分析 → Viseme 时间线 → Live2D 嘴型参数
阶段 3 (可选):     Rive 状态机 + 10+ viseme + 微表情
```

### 6.3 AI 对话 Prompt 架构

参考 gemini-teacher 的思路，单个 LLM 同时完成：

1. **角色扮演**：友好英语外教，语气温和鼓励
2. **自然对话**：根据用户选择的场景/话题展开聊天
3. **实时纠正**：检测语法/用词错误，以自然方式重述（不打断流）
4. **结构化输出**：每条回复包含 `corrections[]` 字段，前端提取后存入错误库
5. **话题引导**：确保覆盖面，适时切换话题，难度递进
6. **复习模式 Prompt**：切换为"课堂模式"，从错误库中提取薄弱点，聊天式复习

## 九、风险与缓解

| 风险 | 缓解措施 |
|------|---------|
| 用户注册 API Key 门槛高 | 新手引导：分步教程 + 推荐供应商（附注册链接）+ 一键测试连接 |
| STT/TTS API 延迟影响体验 | 流式 STT（Deepgram 实时流式）；TTS 预加载 + 本地缓存 |
| 用户 Key 余额不足 | 调用失败时提示友好错误信息，引导检查余额 |
| Flutter Web 音频兼容性 | 早期测试 Safari，准备降级方案 |
| macOS App Store 沙盒审核 | 提前研究 entitlements，阶段四处理 |
| Live2D 模型制作周期 | MVP 用插画 + 振幅动画先跑起来 |
| 不同 LLM 回复格式不一致 | 严格定义 JSON Schema，加 fallback 解析 |
| Glassmorphism 性能开销 | 毛玻璃效果仅用于关键层，低端设备自动降级为半透明纯色 |
| Profile Key 安全 | 系统级 Keychain/加密存储，导出时脱敏 |
