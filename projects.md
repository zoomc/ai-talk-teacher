# SpeakFlow — AI 口语练习应用

> 跨平台 AI 英语口语练习应用
> 更新日期：2026-06-28
> 技术栈：Flutter（Dart）
> 目标平台：macOS · Web · iOS · Android

---

## 一、产品定位

面向英语学习者的 AI 口语练习应用。用户与 AI 虚拟外教进行自然英语对话，AI 在对话中实时纠正语法和用词错误，跟踪学习弱点，并通过复习模式巩固提升。

**核心差异点（对比市面产品）：**
- 用户可自由选择 AI 提供商和模型（不绑定单一服务）
- AI 驱动的"聊天式"复习模式（不是卡片刷题）
- 会话连续性（随时接续上次话题）
- 默认零配置零成本（系统内置 STT/TTS，开箱即用）
- 现代 Glassmorphism 视觉风格，科技感与美感兼具

## 二、竞品参考

| 产品 | 核心亮点 | 值得借鉴 |
|------|---------|---------|
| **Praktika** | 命名AI角色、纠正融入对话流、$8/月 | 纠正策略：不打断对话，自然重述正确表达 |
| **Speak** | OpenAI 投资、说话优先理念 | 3步学习法：学→练→用 |
| **ELSA Speak** | 音素级发音分析、92M下载 | 发音反馈的颗粒度和可视化 |
| **TalkPal** | 10+对话模式（角色扮演、辩论、电话） | 多场景模式设计 |
| **Loora** | 职场英语、TOEFL/IELTS模拟 | 考试准备功能 |

**行业共识：** 成熟产品均使用 viseme 驱动的 2D/轻量 3D 虚拟人物（而非神经网络实时生成），原因：零延迟、60fps、免费、离线可用。

**参考项目 gemini-teacher：** Python 终端应用，用 Gemini 同时处理对话+发音评估+语法纠正，证明单个 LLM 通过精心设计的 prompt 可以同时完成教学和对话。但长期未维护，仅 Gemini 单一提供商。

### 3.2 对话练习模式

| 功能 | 说明 |
|------|------|
| **自由对话** | AI 引导自然英语聊天，自动把控话题覆盖和难度 |
| **场景练习** | 预设场景：餐厅、机场、商务会议、面试、约会等 |
| **智能纠正** | 参考 Praktika 策略——不打断对话流，在回复中自然重述正确表达 |
| **纠正强度** | 三档可调：温和（偶尔纠正）/ 适中 / 严格（每次必纠） |
| **错误自动记录** | 每次纠正自动提取并存储：原文→纠正→类型（语法/用词/发音） |
| **话题连续性** | 每次打开应用可选择：继续上次话题 / 开始新话题 |
| **AI 主导节奏** | LLM 通过 System Prompt 控制对话走向，确保贴近实用场景 |

### 3.3 复习模式

| 功能 | 说明 |
|------|------|
| **AI 引导复习** | 虚拟外教以"聊天式课堂"风格带你复习，语气轻松不严肃 |
| **错误回顾** | 从错误库中提取薄弱点，融入对话重新练习 |
| **间隔重复** | SM-2 算法安排复习间隔，越熟的错误出现频率越低 |
| **进度统计** | 已掌握 / 待复习 / 新增错误的数量和趋势 |

**Profile 管理操作：**

| 操作 | 说明 |
|------|------|
| 新建 | 填写名称 + URL + Key → 获取模型列表 → 选模型 → 保存 |
| 获取模型列表 | 调用 `GET {base_url}/v1/models`，解析返回的模型 ID 列表；失败时允许手动输入模型名 |
| 保存 | 仅保存，不切换当前使用的 Profile |
| 保存并应用 | 保存 + 立即切换为当前活跃 Profile |
| 应用 | 列表中点击，即时生效（对话、TTS 等全部切换到新 Profile） |
| 编辑 | 修改任意字段，可重新获取模型列表，保存后如果当前激活则自动更新 |
| 复制 | 基于现有 Profile 快速创建副本，修改后另存 |
| 删除 | 需确认；**不能删除当前激活的 Profile** |
| 导入/导出 | JSON 格式，方便备份和跨设备迁移（导出时 Key 默认脱敏） |

**协议兼容说明：**
- 所有请求走 `POST {base_url}/v1/chat/completions`，标准 OpenAI 格式
- 模型列表拉取走 `GET {base_url}/v1/models`，标准 OpenAI 格式
- 兼容 DeepSeek、智谱、Kimi、通义千问、MiniMax、OpenRouter、Ollama、火山引擎、百川等全部 OpenAI 协议供应商
- 也兼容任何未列出的第三方中转/代理服务

### 3.5 语音引擎设置

采用"系统默认开箱即用 + 可选云端升级"策略，用户无需任何配置即可开始使用。

**语音识别（STT）：**

| 引擎 | 类型 | 费用 | 用户配置 | 说明 |
|------|------|------|---------|------|
| **系统默认** | 内置 | 免费 | 无需配置 | iOS: Apple Speech; Android: SpeechRecognizer; macOS: Apple Speech; Web: Web Speech API |
| **Deepgram** | 云端 | 按分钟计费 | 需填 API Key | 效果更好，识别率更高，适合进阶用户 |

**语音合成（TTS）：**

| 引擎 | 类型 | 费用 | 用户配置 | 说明 |
|------|------|------|---------|------|
| **系统默认** | 内置 | 免费 | 无需配置 | iOS/macOS: AVSpeechSynthesizer; Android: TextToSpeech; Web: SpeechSynthesis |
| **Fish Audio** | 云端 | 按字符计费 | 需填 API Key | 音质更自然，支持多种音色 |
| **ElevenLabs** | 云端 | 按字符计费 | 需填 API Key | 业界标杆音质，免费额度 ~10000 字符/月 |

**设置界面：**

```
语音识别
  ○ 系统默认（推荐）     ← 零配置，免费，离线可用
  ○ Deepgram            ← 效果更好，需注册获取 API Key
     API Key: [ sk-******** ]  ← 选中云端时才显示

语音合成
  ○ 系统默认（推荐）     ← 零配置，免费
  ○ Fish Audio          ← 更自然的语音，需注册获取 API Key
     API Key: [ ********** ]   ← 选中云端时才显示
  ○ ElevenLabs          ← 顶级音质，有免费额度
     API Key: [ ********** ]
```

**设计原则：**
- 默认体验 = 零配置、零成本、四端可用
- 云端引擎 = 可选升级，用户自行注册获取 Key
- 不暴露技术细节（不提"本地模型""ONNX"等概念）
- 切换引擎即时生效，无需重启

## 四、UI 设计规范

### 4.1 设计语言

采用 **Glassmorphism + 平台自适应** 策略，统一视觉风格的同时尊重各平台原生体验。

**核心视觉元素：**

| 元素 | 说明 |
|------|------|
| 毛玻璃背景 | 半透明磨砂材质，营造层次感和深度 |
| 微渐变 | 紫→蓝→青冷色调渐变，科技感与学习感兼具 |
| 柔光效果 | 关键元素（按钮、卡片）带柔和外发光，暗示 AI 智能感 |
| 大圆角 | 16-24px 圆角卡片，友好不生硬 |
| 深色优先 | 默认深色主题，护眼 + 科技氛围；支持浅色模式 |
| 流体动画 | 状态切换、页面转场使用弹性动画，不突兀 |

**色彩方案：**

```
主色调（深色模式）：
  背景：    #0A0E1A → #111827（深空蓝渐变）
  玻璃层：   rgba(255,255,255, 0.06) ~ 0.12
  主强调色：  #6C5CE7（紫色，AI/科技感）
  次强调色：  #00D2FF（青色，活力感）
  成功/纠正： #00E676（绿色）
  错误标注：  #FF5252（红色，用于错误词高亮）
  文字主色：  #F0F0F0
  文字次色：  #8892A4

浅色模式：
  背景：    #F5F7FB → #FFFFFF
  玻璃层：   rgba(255,255,255, 0.7) ~ 0.9
  强调色同深色模式，饱和度微调
```

### 4.2 平台适配策略

| 平台 | 风格实现 | 说明 |
|------|---------|------|
| **iOS / macOS** | Liquid Glass 风格 | 追随 iOS 26 设计语言：系统级毛玻璃材质、动态透明度、原生导航手势。使用 `BackdropFilter` + 系统材质 API |
| **Android** | Material You + 玻璃质感 | Material 3 组件为基础，叠加毛玻璃效果。支持 Dynamic Color 动态取色（壁纸提取主题色） |
| **Web** | 与所在平台风格统一 | macOS Safari 用 Liquid Glass 风格，Windows Chrome 用 Material 风格 |

**适配原则：**
- 核心组件（聊天气泡、AI 角色区域、录音按钮）视觉风格四端统一
- 导航栏、弹窗、系统交互遵循各平台原生规范
- 玻璃效果通过 `BackdropFilter`（Flutter 原生支持）实现，Web 端用 `backdrop-filter` CSS

## 五、数据架构

### 5.1 本地存储方案

| 数据类型 | 存储方式 | 说明 |
|---------|---------|------|
| **Profile 配置** | SQLite | 名称、URL、加密后的 Key、模型名 |
| **API Key** | 系统 Keychain | iOS/macOS: Keychain; Android: EncryptedSharedPreferences; Web: 加密 localStorage |
| **对话历史** | SQLite | 会话 ID、消息列表、时间戳、话题标签 |
| **错误记录** | SQLite | 原文、纠正、类型、时间、复习次数、下次复习时间 |
| **TTS 音频缓存** | 本地文件系统 | 哈希索引，避免重复合成 |
| **Viseme 时间线** | JSON 文件 | 与 TTS 音频配对，唇形同步用 |
| **用户设置** | SQLite / SharedPreferences | 非敏感配置 |

### 5.2 Profile 数据安全

参考 cc-switch 的设计：
- **原子写入**：写入临时文件 → 重命名替换，防止写入中断导致损坏
- **自动备份**：保留最近 5 份 Profile 数据备份
- **Key 加密**：API Key 存入系统级安全存储，不以明文写入数据库
- **导出脱敏**：导出 Profile 时 Key 字段默认掩码处理

## 六、技术方案概要

### 6.1 框架与关键依赖

| 层面 | 方案 |
|------|------|
| **跨平台框架** | Flutter 3.x + Dart |
| **AI 对话** | OpenAI 兼容协议，用户自选提供商 |
| **STT** | 系统默认（Apple Speech / Android SpeechRecognizer / Web Speech API）；可选升级：Deepgram |
| **TTS** | 系统默认（AVSpeechSynthesizer / Android TTS / Web SpeechSynthesis）；可选升级：Fish Audio / ElevenLabs |
| **UI 风格** | Glassmorphism + 平台自适应（iOS: Liquid Glass; Android: Material You + 玻璃质感） |
| **唇形同步** | MVP：振幅驱动嘴型；生产：Live2D + Rhubarb viseme |
| **动画引擎** | Live2D Cubism SDK / Rive / Flutter 动画 |
| **音频** | `record`（录音）+ `just_audio`（播放） |
| **数据库** | `sqflite`（SQLite） |
| **状态管理** | Riverpod 或 Bloc |
| **路由** | GoRouter |
| **加密** | `flutter_secure_storage` |
| **玻璃效果** | `BackdropFilter` + 系统材质 API |

### 6.2 唇形同步方案

```
阶段 1 (MVP):     音频振幅 → 3 状态嘴型（闭/半开/全开）→ 2D 插画
阶段 2 (生产):     TTS 音频 → Rhubarb 离线分析 → Viseme 时间线 → Live2D 嘴型参数
阶段 3 (可选):     Rive 状态机 + 10+ viseme + 微表情
```

### 6.3 AI 对话 Prompt 架构

参考 gemini-teacher 的思路，单个 LLM 同时完成：

1. **角色扮演**：友好英语外教，语气温和鼓励
2. **自然对话**：根据用户选择的场景/话题展开聊天
3. **实时纠正**：检测语法/用词错误，以自然方式重述（不打断流）
4. **结构化输出**：每条回复包含 `corrections[]` 字段，前端提取后存入错误库
5. **话题引导**：确保覆盖面，适时切换话题，难度递进
6. **复习模式 Prompt**：切换为"课堂模式"，从错误库中提取薄弱点，聊天式复习

## 九、风险与缓解

| 风险 | 缓解措施 |
|------|---------|
| Flutter Web 音频兼容性 | 早期测试 Safari，准备降级方案 |
| macOS App Store 沙盒审核 | 提前研究 entitlements，阶段四处理 |
| Live2D 模型制作周期 | MVP 用插画 + 振幅动画先跑起来 |
| 不同 LLM 回复格式不一致 | 严格定义 JSON Schema，加 fallback 解析 |
| 系统 TTS 音质一般 | 可接受——默认体验优先可用性；云端 TTS 作为升级选项 |
| 各平台 STT API 差异 | 封装统一接口，按平台适配，降级处理 |
| Glassmorphism 性能开销 | 毛玻璃效果仅用于关键层，低端设备自动降级为半透明纯色 |
| Profile Key 安全 | 系统级 Keychain/加密存储，导出时脱敏 |
